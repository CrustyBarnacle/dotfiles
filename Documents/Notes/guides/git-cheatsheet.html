git — log review, lost commits & abandoned branches

Not GitHub — plain git. Everything here works offline, in any local repo. Nothing is ever quite as gone as it looks.

Tier 1 — Survival
Status & staging
git statuswhat's changed, staged, untracked
git add <file>stage a file
git add -pstage hunks interactively
git restore --staged <file>unstage a file
git commit -m "msg"commit staged changes
git commit --amendedit the last commit (rewrites its hash)
Diffing
git diffunstaged changes vs. index
git diff --stagedstaged changes vs. last commit
git diff <a> <b>diff between two commits/branches
git diff HEAD~3diff vs. 3 commits ago
Basic log
git logfull commit history, newest first
git log --onelineone line per commit
git log -n 5last 5 commits only
git show <hash>full diff of one commit
Undo (safe, local, uncommitted)
git restore <file>discard uncommitted changes to a file
git checkout -- <file>same, older syntax
git clean -npreview untracked files that would be deleted
git clean -fdactually delete untracked files/dirs
Tier 2 — Everyday
Branches
git branchlist local branches
git branch -alist local + remote branches
git switch <branch>change branch
git switch -c <name>create + switch to new branch
git branch -d <name>delete branch (safe, checks merged)
git branch -D <name>force-delete branch (unmerged commits go dangling, not gone — see Tier 4)
git merge <branch>merge branch into current
Stash
git stashshelve uncommitted changes
git stash listlist all stashes
git stash popreapply + remove most recent stash
git stash apply stash@{n}reapply without removing
git stash drop stash@{n}delete a stash entry
Remotes
git remote -vlist remotes and their URLs
git fetchdownload remote refs, no merge
git pullfetch + merge current branch
git pushpush current branch
git push -u origin <branch>push + set upstream tracking
Filtering the log
git log --oneline --graph --allASCII graph of every branch
git log --author="name"only commits by an author
git log --since="2 weeks ago"time-bounded log
git log -- <path>only commits touching a path
git log -S"text"pickaxe: commits that added/removed a literal string
git log -G"regex"same idea, but regex against the diff
Tier 3 — Power: history archaeology
Full-history sweeps
git log --all --oneline --graphevery ref, not just current branch
git log --all --sourcesame, but shows which ref each commit is from
git log --all --full-history -- <path>every version of a file, even if later deleted
git log --follow -- <path>follow a file's history across renames
git notes — invisible to plain log
git notes listlist every note and the commit it's attached to
git notes show <commit>read the note on a specific commit
git log --show-notesshow notes inline while browsing history
Notes live in refs/notes/commits — a separate ref tree that git log ignores unless asked.
Inspecting stash contents
git stash listsee every stash, even old ones
git stash show -p stash@{0}full diff of a stash
git show stash@{0}same, via git show
A stash is a real (if unusual) commit — git log on its own won't show it, but git show/git cat-file will.
blame & bisect
git blame <file>who last touched each line, and when
git blame -L 10,20 <file>blame just a line range
git bisect startbegin binary-search for a bad commit
git bisect good/bad <sha>mark a commit while bisecting
git bisect resetend bisect, return to original HEAD
Tier 4 — "Where'd I put that?!" recovery
Reflog — your real undo button
git reflogevery place HEAD has pointed, most recent first
git reflog show --allreflogs for every ref, not just HEAD
git reflog show <branch>reflog for one specific branch
git checkout <sha-from-reflog>jump to any prior state
Reflog is local-only and expires (default 90 days) — but it survives resets, rebases, and branch deletion.
Dangling & unreachable objects
git fsck --unreachablelist commits/blobs/trees no ref points to
git fsck --full --unreachable --danglingdeeper sweep, includes reflog-only objects
git cat-file -p <hash>pretty-print any object by its hash
git cat-file -t <hash>show an object's type (blob/tree/commit)
git show <hash>works on dangling commits too, if you have the hash
Recovering a deleted branch
git reflog | grep checkoutfind the last time you were on the deleted branch
git branch <name> <sha>recreate a branch pointer at that commit
git log --walk-reflogsbrowse history through reflog entries directly
After a bad reset --hard
git reflogfind the sha you were at before the reset
git reset --hard <old-sha>go back to it
git cherry-pick <sha>or just replay one lost commit onto current branch
Recovering a deleted file's old content
git log --all --full-history -- <path>find every commit that touched it
git show <sha>:<path>print the file as it was at that commit
git checkout <sha> -- <path>restore that version into your working tree
Object plumbing reference
git cat-file -p <hash>pretty-print blob/tree/commit contents
git cat-file -t <hash>type of an object
git ls-tree -r <hash>recursively list a tree's contents
git rev-parse <ref>resolve any ref/shorthand to its full hash
Guided exercises — practice in any throwaway repo
Exercises in order
01
git log --all --oneline --graph
Run this in any repo with more than one branch. Compare it to plain git log — notice how much more shows up.
02
git stash; echo "hi" >> some_file; git stash; git stash list
Create two stashes, then inspect one with git stash show -p stash@{0} before popping or dropping it.
03
git notes add -m "test note" HEAD~1 && git notes show HEAD~1
Attach a note to an old commit, then confirm plain git log doesn't mention it at all.
04
git switch -c throwaway && git commit --allow-empty -m "temp" && git switch main && git branch -D throwaway
Deletes the branch, but not the commit. Now find it: git reflog | grep throwaway.
05
git fsck --unreachable
Run right after exercise 04 — the orphaned commit from the deleted branch should show up as an unreachable commit.
06
git cat-file -p <hash-from-05>
Read the dangling commit directly by hash, no branch required. This works even after the branch ref is long gone.
07
git branch recovered <hash-from-05>
Turn the dangling commit back into a real, reachable branch — fully "un-deleting" it.
08
git log -S"some_unique_string" --all
Pickaxe search: find the exact commit that introduced or removed a specific string, anywhere in history.
The single best habit to build first: git log alone shows a tiny slice of what's actually in a repo. Notes, stashes, dangling commits, and reflogs are all real, readable data — they just need their own command to surface. Almost nothing in git is truly gone until git gc runs (and even then, rarely immediately).
